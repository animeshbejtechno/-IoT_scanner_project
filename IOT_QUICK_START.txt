🔧 REMEDIATION STEPS

For Telnet/FTP Access:
  1. SSH into device: ssh admin@device_ip
  2. Disable Telnet: systemctl disable telnetd
  3. Test: ssh -v admin@device_ip

For Default Credentials:
  1. Access device via admin panel
  2. Go to Settings → Security
  3. Change username and password
  4. Verify with new credentials

For Outdated Firmware:
  1. Visit manufacturer support page
  2. Download latest firmware
  3. Boot device into recovery mode
  4. Upload and flash firmware
  5. Verify version: Check System Info

For HTTPS Configuration:
  1. Generate SSL certificate
  2. Configure service to use HTTPS
  3. Redirect HTTP to HTTPS
  4. Test: https://device_ip
